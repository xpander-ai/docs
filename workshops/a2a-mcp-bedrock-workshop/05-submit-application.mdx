## ğŸ•¹ï¸ Working with Your Manager Agent via the Web UI

The manager agent is your main point of contact for building applications with your agent fleet. You'll interact with it through the xpander.ai web UI:

1. Go to https://app.xpander.ai and go to your manager AI Agent (AI Development Manager)
2. In the canvas, click the "Chat" source node and click the URL to open the Chat UI.
3. Enter a prompt describing the application you want to build and instruct the manager agent to submit it when ready. For example:

```bash example prompt
Build a minimal â€œQuickTodoâ€ single-page app using vanilla HTML, CSS, and JavaScript. Users must be able to add, edit, and delete tasks; data should persist in localStorage so tasks remain after reload. Acceptance criteria: adding shows the new task immediately, editing updates it in place, deleting removes it, and a page reload restores all tasks. Code must pass ESLint. Target branch: may-2025/my-github-username/quick-todo
```

<Note>
Change the <my-github-username> part to your GitHub username
</Note>
  
The manager agent will interpret your request, coordinate the developer agents, and handle the submission process for you.

Feel free to get creative\! The manager agent will break down your request and assign tasks to the developer agents.

<Note>
  Your manager agent will coordinate the work between your developer agents, review their code, and select the best implementation. This all happens automatically thanks to the Agent-to-Agent (A2A) capabilities you configured.
</Note>

## ğŸ“¡ Monitor Agent Collaboration with xpander logs

If you want to see detailed logs of agent collaboration and task execution:

1. In your terminal, navigate to both your manager and developer agent folders.
2. Run the following command in each folder:

```
xpander logs
```

This will let you see how the manager agent triggers the developer agents, delegates tasks, and reviews their work in real time.
