## Overview

This is the simplest possible hello world example using the xpander.ai SDK with Agno framework integration. The Backend class automatically provides all agent settings from the xpander.ai platform, making it incredibly easy to get started.

## Quick Start

### Prerequisites

```bash
pip install "xpander-sdk[agno]"
```

### Environment Setup

Create a `.env` file with your API keys:

```bash
XPANDER_API_KEY=your_xpander_api_key
XPANDER_ORGANIZATION_ID=your_organization_id
XPANDER_AGENT_ID=your_agent_id
OPENAI_API_KEY=your_openai_api_key  # or other LLM provider

```

### Complete Hello World Agent

Create a file named `hello_world.py`:

```python
from xpander_sdk import Backend, on_task, Task
from agno.agent import Agent

@on_task
async def on_task(task: Task):
    # Get all agent settings from xpander.ai platform
    backend = Backend()
    
    # Create Agno agent with platform settings
    agno_agent = Agent(
        **backend.agno_backend_args()
    )
    
    # Process the task using Agno
    result = await agno_agent.arun(message=task.to_message())
    
    # Set the result
    task.result = result.content
    
    return task
```

That's it! Run your agent:

```bash
python hello_world.py
```

## How It Works

<Steps>
  <Step title="Task Reception">
    The `@on_task` decorator automatically listens for incoming tasks from the xpander.ai platform
  </Step>
  <Step title="Backend Integration">
    `Backend().get_args()` fetches all agent configuration (instructions, tools, knowledge bases) from your xpander.ai agent
  </Step>
  <Step title="Agno Processing">
    The Agno agent processes the task using the platform configuration with your chosen LLM
  </Step>
  <Step title="Result Return">
    The processed result is automatically sent back to the platform
  </Step>
</Steps>

## Key Components

### Backend Class
- **Purpose**: Provides seamless integration with xpander.ai platform
- **Method**: `get_args(override={})` - Fetches agent settings with optional overrides
- **Benefits**: No manual configuration needed - everything comes from the platform

### Task Object
- **Properties**: Contains input data, files, user context, and metadata
- **Method**: `to_message()` - Converts task to Agno-compatible message format
- **Result**: Set `task.result` to return processed output

### @on_task Decorator
- **Function**: Automatically handles task lifecycle and event listening
- **Requirements**: Function must accept `task` parameter and return the task
- **Flexibility**: Supports both sync and async functions

## Customization Options

### Override Model Settings

```python
from agno.models.anthropic import Claude

@on_task
async def on_task(task: Task):
    backend = Backend()
    
    agno_agent = Agent(
        **backend.get_args(override={
            "model": Claude(id="claude-3-sonnet-20240229"),
            "temperature": 0.7
        })
    )
    
    result = await agno_agent.arun(message=task.to_message())
    task.result = result.content
    
    return task
```
